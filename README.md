# Mycelium

**Coordinating urban farmers like mycelium coordinates a forest — each node contributing what it does best, the whole network thriving because of it.**

---

## The Problem

Food is expensive. A university student tracking their spending for the first time found a startling amount going to fast food — McDonald's runs adding up week after week. The frustrating part: they remembered growing food in a backyard garden as a child for a fraction of the cost. The food was fresher, healthier, and nearly free.

Urban farming is the obvious answer. But two barriers stop most people from making it work:

**1. Knowledge.** Most people don't know what to plant, when to plant it, how to care for it, or what conditions their plot actually needs. Growing food without guidance produces poor yields and discouragement.

**2. Effectiveness.** Even if you figure it out, growing at home alone you're limited to one or two plants. You still have to go to the grocery store for everything else. A single tomato plant doesn't solve your food budget.

The result: urban farming stays a hobby rather than becoming a real alternative to expensive food.

---

## The Solution

Mycelium operates at the **community level**, not the individual level.

Instead of one person trying to grow a little of everything, Mycelium coordinates a network of urban farmers so that each person grows what they are best suited to grow — and the collective output covers everything the community needs. It applies the same economic principle behind industrial food production — **economies of scale and comparative advantage** — to urban farming networks.

A single farmer growing tomatoes on a balcony is a hobby. A thousand coordinated urban farmers growing complementary crops across a city is a food system.

---

## How It Works

### The Nodal Network

Every farmer is a **node**. Nodes enter the network by submitting their plot data — size, soil conditions, tools, location, preferences. The network is designed for easy entry and exit: joining takes minutes, and farmers can pause or leave without breaking the system.

### The Optimization Engine

An ILP (integer linear programming) optimizer runs across the full network state, assigning each farmer the crop they are comparatively best suited to grow, while filling the gaps the network actually needs. No two farmers end up growing the same surplus. Every node contributes something distinct.

### AI-Powered Growing Guidance

Once assigned, farmers receive a personalized task list generated by **Gemini 2.5 Flash** — specific to their crop, their plot size, their hub's priority, and their current cycle. Instead of generic advice, farmers get precise instructions: what to do, when, and why. Knowledge is no longer a barrier.

### Distribution Hubs

Farmers deliver their harvest to local **distribution hubs**. Hub managers oversee inventory, approve incoming requests, and coordinate pickups. The hub network ensures food reaches the community, not just the individual grower.

### The Ledger

Every delivery and pickup is recorded in a ledger. Farmers earn credit for what they produce; community members spend credit to receive food. The economy is backed by real production — fair, auditable, and self-balancing.

## Key Features

- **Live network map** — admin view of all farms, hubs, routing, and network state
- **Crop optimizer** — ILP-based assignment maximizing yield across the full network
- **AI task generation** — Gemini 2.5 Flash generates personalized growing tasks per farmer per cycle, delivered as background tasks so the UI never blocks
- **Hub management** — create/delete hubs, generate access keys, manage inventory and requests
- **Hub key auth** — hub managers authenticate with a unique key; demo mode available for testing
- **MyFood farmer app** — mobile-friendly farmer interface with task lists, cycle tracking, and request management
- **Admin dashboard** — nodal network map, ledger, data explorer, hub manager, optimizer controls

## Getting Started

### Backend

```bash
cd app/backend
pip install -r requirements.txt
uvicorn app.backend.api.main:app --reload
```

Requires a `.env` file with:
```
GEMINI_API_KEY=...
VITE_GOOGLE_MAPS_API_KEY=...
VITE_MAP_ID=...
```

### Frontend

```bash
cd app/frontend
npm install
npm run dev
```
